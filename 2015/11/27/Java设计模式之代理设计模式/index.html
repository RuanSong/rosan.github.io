Java设计模式之代理设计模式

  • Java
  • Proxy
  • 代理设计模式
  • 设计模式

为某个对象提供一个代理，以控制对这个对象的访问。 代理类和委托类有共同的父类或父接口，这样在任何使用委托类对象的地方都可以用代理对象替代。代理类负责请求的预处理、过滤、将请求分派给委托类处理、以及委托类执行完请求后的后续处理。

代理的组成

在代理模式当中，我们需要定义三个角色：
1.抽象角色:通过接口或者抽象类声明真是角色的业务逻辑方法
2.代理角色:实现抽象角色，是真实角色的代理，通过真实角色的业务逻辑方法来实现抽象类。
3.真实角色:实现抽象角色,定义真实角色所要实现的业务逻辑，提供给代理角色调用。

代理的分为静态代理和动态代理

代理模式的使用场景

1.一个对象，一副很大的图像，需要载入很长时间
2.一个运行算，需要很长时间才能完成的计算结果，并且要在计算过程中显示运算结果
3.权限验证，在执行代理之前先验证用户的权限相关信息。

动态代理

动态代理的源码是程序在运行期间有JVM根据反射机制动态生成的，因此代理类和抽象实现类的关系在程序运行的时候才确定。
